起点バグ「究極の二択プリセットが遊べない」を軸に、同系統の不具合を全域調査・修正してください。

## アプリ概要

- **PartyRant-jp**: 大人向けパーティーゲームアプリ（合コン・飲み会・キャバクラ等）
- **スタック**: Next.js 14 App Router / TypeScript / Tailwind CSS v4 / Supabase（Broadcast + Postgres）
- **リポジトリ**: `Y:/webwork/PartyRant-jp`（同じSupabaseDBを`Y:/webwork/PartyRant-fam`と共有）

### ゲームモード

| mode | 説明 |
|------|------|
| `trivia` | 正解あり早押しクイズ、スコアランキング |
| `polling` | 正解なし投票・実態調査、VoteBar表示 |
| `opinion` | 少数派/多数派どちらが負けか宣言型 |

### 主要な画面フロー

```
/presets → ゲーム設定 → /api/games（POST） → /play/[gameId]（ホスト）
                                              /join/[code]（ゲスト）
```

### プリセットのカテゴリー（scene フィールド）

- `みんなで` / `多数派クイズ` / `究極の二択` / `この中で●●なのは誰だ`
- `結婚式` / `合コン` / `キャバクラ` / `会社飲み会` / `ホームパーティー` / `サークル` / `居酒屋`
- `雑学クイズ`

### 名前置換システム（JP固有）

- プリセット問題の選択肢に `プレイヤーA` `プレイヤーB` `プレイヤーC` `プレイヤーD` が含まれる場合、
  ロビーで入力された参加者名に置き換えてゲームを開始する
- `src/app/api/games/[gameId]/advance/route.ts` の lobby→question 遷移時に置換処理を実行
- SSEイベント `question_started` に `game: finalGame`（置換済みゲームオブジェクト）を含めてブロードキャスト
- ゲスト側（`GuestGameClient.tsx`）は `question_started` イベントで `event.game` があれば `setGame(event.game)` する

### もう一度フロー（SSEベース）

- ホストが「🔄 もう一度」→ POST `/api/games/[gameId]/next` → 新ゲーム作成 →
  旧チャンネルに `{ type: 'next_game', joinCode }` をブロードキャスト →
  ゲストは自動で `/join/[newCode]` にリダイレクト

## 起点バグ：究極の二択が遊べない

### 症状

`/presets` ページで scene=`究極の二択`（mode=`polling`）のプリセットを選んでゲームを開始しようとすると、
遊べない（エラー、または開始ボタンが機能しない）。

### 調査すべき箇所

1. **`src/app/presets/page.tsx`**
   - `究極の二択`（`mode: polling`）プリセットの「はじめる」ボタンがどのAPIを呼ぶか確認
   - `polling` モードの場合に設定モーダルがどう動くか（`SettingsMode` 型が `trivia` のみになっていないか）
   - 直接プリセットを起動するパス（`/api/games` への POST）でどんな body を送るか
   - 問題数（`count`）が指定されている場合、`polling` モードで問題を絞り込む処理があるか

2. **`src/app/api/games/route.ts`（POST）**
   - `polling` モードのプリセット起動時に渡されたパラメータが正しく処理されるか
   - `questions` の slice/sample 処理が `polling` で問題を起こしていないか
   - `mode` や `loseRule` のバリデーションで弾かれていないか

3. **`src/app/api/games/[gameId]/advance/route.ts`**
   - `polling` モードでゲームが `lobby → question` に進めるか
   - `question_started` ブロードキャストに `game: finalGame` が含まれているか（含まれていないと
     ゲスト側で名前置換が反映されない）

4. **`src/app/play/[gameId]/PlayGameClient.tsx`**
   - ホスト画面で `polling` モード時の表示・進行フローに欠落がないか
   - `mode === 'polling'` かつ `status === 'question'` のフェーズが正しく表示されるか

## 同系統バグの調査リスト

### A. プリセット起動フロー全般

- [ ] `trivia` / `polling` / `opinion` の3モードそれぞれで「はじめる」ボタンが正しく動くか
- [ ] 問題数選択（5/10/15問）が `polling` / `opinion` モードのプリセットに対して正しく機能するか
      （`polling` のプリセットは全問使いたいケースが多い — 10問指定で20問プリセットを削ると意図が変わる）
- [ ] `loseRule`（`minority` / `majority`）が `opinion` モードのゲームに正しく引き継がれるか

### B. 名前置換システム

- [ ] `advance/route.ts` でロビー→問題遷移時、置換後の `game` を SSE に含めているか
- [ ] ゲスト側 `GuestGameClient.tsx` で `question_started` イベントの `event.game` を受け取って
      `setGame` しているか（していないと選択肢がABCのまま）
- [ ] 置換後に `store.updateGameQuestions(gameId, substitutedQuestions)` を呼び、
      その後 `store.getGame(gameId)` で最新を取り直してから SSE ブロードキャストしているか
- [ ] 名前が4人未満の場合（プレイヤーA〜Cしかいないのに選択肢が4つ等）の置換漏れがないか

### C. SSEイベント取りこぼし

- [ ] `question_started` に続く `question_ended` `game_updated` 等のイベント型が
      `src/lib/events/types.ts` に全て定義されているか
- [ ] ゲスト側がSSE接続前に届いたイベントを初回ロード時のHTTPポーリングで補完できているか
- [ ] `next_game` イベント受信時、ゲストが `/join/[joinCode]` にリダイレクトする処理が
      `ended` 状態だけでなく `waitingForNext` 等の中間状態でも機能するか

### D. ホスト参加モード（`hostParticipating`）

- [ ] ホスト参加ボタン（`ホストも参加する`）が `PlayGameClient.tsx` に存在し、
      ロビー状態で正しく表示されるか（以前、`HostGameClient.tsx` という使われないファイルに
      誤って実装されたことがあった — `src/app/host/[gameId]/page.tsx` が `redirect('/host')` しているため
      `HostGameClient.tsx` は一切レンダリングされない）
- [ ] ホストが回答送信後、`hostAnswered` が `true` になり VoteBar が表示されるか
- [ ] ホスト参加時のスコアが最終リーダーボードに反映されるか（`trivia` モード）

### E. ゲーム終了後のランキング表示

- [ ] `polling` モードのゲーム終了後、ホスト画面（`PlayGameClient.tsx`）に
      `みんなの実態まとめ`（多数派/少数派ランキング）が表示されるか
- [ ] 選択肢が全員プレイヤー名のとき（`this中で●●なのは誰だ`系）に
      `選ばれた回数`ランキングが表示されるか
- [ ] 同ランキングがゲスト画面（`GuestGameClient.tsx`）にも表示されるか

### F. もう一度フロー

- [ ] ホストが「🔄 もう一度」→ `POST /api/games/[gameId]/next` → 新ゲームの `joinCode` を
      ゲストにブロードキャストするフローが機能するか
- [ ] `next_game` SSEイベントを受信したゲストが自動リダイレクトするか
- [ ] 新ゲームのロビーに遷移後、元のゲストが正しく表示されるか（`/join/[newCode]` の初期ロード）

## 修正方針

- **最小限の変更**で確実に直す。動いている機能を壊さない
- `polling` / `opinion` モードのプリセット起動フローが `trivia` と同等に機能することを目標とする
- 型安全性を維持する（`as` キャストや `!` アサーションの乱用を避ける）
- 修正後は該当フローを手動で通して確認できる手順をコメントで残す
